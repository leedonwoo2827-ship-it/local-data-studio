# 대시보드 설계 원칙 (Green/Red 신호 기준 포함)

참조: 대학 자율성과지표 대시보드(상단 도넛 + 달성도 현황판 + 지표별 신호).

## 레이아웃 원칙
1. **위에서 아래로 추상→구체**: ① 전체 비율(도넛) → ② KPI 카드(개별 신호) → ③ 달성도 표 → ④ 추이 차트.
2. **신호 우선**: 색(Green/Red)으로 30초 안에 "어디가 문제인지" 보이게.
3. **목표를 항상 같이**: 실적만 보여주지 말고 목표·달성률을 함께(맥락 없는 숫자는 판단 불가).
4. **한 화면 = 한 팀**: 비교는 기간 필터/팀 전환으로. 한 화면에 모든 팀을 욱여넣지 않음.

## Green/Red 신호 기준
- 달성비율 = higher_better: `실적/목표`, lower_better: `목표/실적`.
- **Green** = 달성비율 ≥ `green_threshold`(기본 0.9), **Red** = 미만.
- 임계값 선택:
  - `1.0` — 엄격(목표 100% 달성해야 Green). 평가·심사용.
  - `0.9` — 표준(90% 이상이면 양호). 운영 모니터링용. ← 본 샘플 기본.
  - `0.8` — 관대(초기 단계/변동 큰 지표).
- "노랑(주의)"이 필요하면 향후 `yellow_threshold` 도입 여지(현재는 2단계).

## 보고서(중간/최종)와의 연결
- **실시간 대시보드** = 지금 상태 모니터링.
- **중간보고서** = 남은 기간 리스크와 조정안(진행형).
- **최종보고서** = 종합 평가와 차기 제언(완료형).
- 셋 다 같은 KPI 계산 결과를 공유 → 숫자 불일치가 없다(단일 진실원천).

## 안티패턴 (피하기)
- 축 0 미시작 막대(왜곡), 3D 파이, 너무 많은 색.
- 목표 없는 KPI(달성 여부 판단 불가).
- 정의가 모호한 합산(비율을 sum 하는 등) → `agg` 를 지표 성격에 맞게.
